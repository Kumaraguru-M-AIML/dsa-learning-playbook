# ⚒️ THE FORGE PROTOCOL
### *Framework for Organized Reality-Grounded Execution*

> **"Raw knowledge is ore. You forge it into weapons."**
> Every book, paper, video, idea, experiment you encounter is ore.
> FORGE is the process that turns ore into tools you actually use.

---

## 🧠 THE CORE PHILOSOPHY

Most people consume knowledge and **accumulate** it.
FORGE makes you consume knowledge and **deploy** it.

The difference:
- Accumulation → Noting what's interesting
- Deployment → Knowing *when* and *how* to use each thing

You don't need more information. You need **better weapons**.

---

## ⚙️ THE 5-STAGE FORGE PROCESS

```
FIND → ORGANIZE → ROUTE → GENERATE → EXECUTE
  F       O          R        G          E
```

| Stage | Name | What You Do | Output |
|-------|------|-------------|--------|
| **F** | **Find** | Go through a source. Ask: *"Does this change what I do or how I think?"* | Highlights / raw captures |
| **O** | **Organize** | Convert each capture into one Atomic Note. One idea, one note, one title as a claim. | Atomic Notes (in `01_Atomic_Notes/`) |
| **R** | **Route** | Link each Atomic Note to related concepts, contradictions, or synergies. | Connected network of notes |
| **G** | **Generate** | Pull related Atomic Notes → synthesize into a Mental Model. Answer: *"What do I now believe + what will I do differently?"* | Mental Models (in `02_Mental_Models/`) |
| **E** | **Execute** | Translate a Mental Model into a Playbook — a checklist or protocol you actually run. | Playbooks (in `03_Playbooks/`) |

## 📏 THE RULES OF FORGE

### Rule 1: The "So What?" Filter
Before creating an Atomic Note, ask: *"So what? How does this change what I do?"*
If you can't answer that — skip it. You don't need it.

### Rule 2: One Note = One Claim
An Atomic Note title is always a **statement**, not a topic.
- ❌ BAD title: `"Testosterone"`
- ✅ GOOD title: `"High dietary fat directly raises free testosterone"`

### Rule 3: The Deploy Condition
Every Atomic Note must have a **When to use** field.
*"I reach for this when..."* — if you can't write this, the note isn't finished.

### Rule 4: Connect or It Dies
A note with zero links to other notes is a dead note. After creating an Atomic Note, you must link it to at least **one other note**.

### Rule 5: Model Before Playbook
You don't write a Playbook until you have at least 3 Atomic Notes feeding into a Mental Model. No shortcuts.

### Rule 6: Weekly Synthesis
Once a week: review new Atomic Notes → look for patterns → generate or update a Mental Model.

## 🔄 THE READING WORKFLOW (Step by Step)

### PHASE 1: BEFORE reading
1. Open `05_Source_Queue/SOURCE_QUEUE.md`
2. Pick the source
3. Set an **intention**: *"What am I trying to learn or solve right now?"*
4. Write it at the top of a scratch note.

### PHASE 2: WHILE reading
1. As you read, highlight or bracket anything that hits you
2. Next to each highlight, write in ONE sentence: *why this matters to me*
3. Use shorthand tags in the margin: `[C]` = concept, `[T]` = technique, `[!]` = urgent, `[?]` = need to verify

### PHASE 3: AFTER each reading session
1. Go through your highlights
2. For each one worth keeping → create an **Atomic Note** (use `_TEMPLATE_Atomic.md`)
3. Link it to any existing notes it connects to

### PHASE 4: AFTER finishing the source
1. Look at all Atomic Notes from this source
2. See if a **Mental Model** emerges → if so, write it (use `_TEMPLATE_Model.md`)
3. Update `04_Concept_Map/FORGE_CONCEPT_MAP.md`
4. If a Model has enough richness → write a **Playbook** (use `_TEMPLATE_Playbook.md`)
5. Mark the source as `✅ FORGED` in the Source Queue
